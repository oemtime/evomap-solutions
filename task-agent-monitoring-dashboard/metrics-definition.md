# Agent监控仪表盘 - 关键指标定义

## 1. 核心性能指标 (Core Performance Metrics)

### 1.1 响应时间指标
| 指标名称 | 说明 | 目标值 | 告警阈值 |
|---------|------|--------|----------|
| `avg_response_time` | 平均响应时间 | < 2s | > 5s |
| `p50_response_time` | 50分位响应时间 | < 1s | > 3s |
| `p95_response_time` | 95分位响应时间 | < 3s | > 8s |
| `p99_response_time` | 99分位响应时间 | < 5s | > 10s |
| `max_response_time` | 最大响应时间 | < 10s | > 15s |

### 1.2 成功率指标
| 指标名称 | 说明 | 目标值 | 告警阈值 |
|---------|------|--------|----------|
| `success_rate` | 请求成功率 | > 99% | < 95% |
| `error_rate` | 错误率 | < 1% | > 5% |
| `timeout_rate` | 超时率 | < 0.5% | > 2% |
| `retry_rate` | 重试率 | < 5% | > 10% |

### 1.3 Token使用指标
| 指标名称 | 说明 | 目标值 | 告警阈值 |
|---------|------|--------|----------|
| `tokens_per_request` | 单次请求Token数 | < 2000 | > 4000 |
| `prompt_tokens` | Prompt Token数 | - | - |
| `completion_tokens` | Completion Token数 | - | - |
| `cost_per_request` | 单次请求成本 | < $0.01 | > $0.05 |
| `daily_cost` | 日累计成本 | < $100 | > $200 |

## 5. 告警级别定义

### 5.1 告警等级
- **P0 - Critical**: 系统不可用或核心功能故障，立即处理
- **P1 - High**: 严重影响用户体验，1小时内处理
- **P2 - Medium**: 部分功能受影响，4小时内处理
- **P3 - Low**: 轻微影响，24小时内处理

### 5.2 告警渠道
- P0: 电话 + 短信 + 邮件 + Slack
- P1: 短信 + 邮件 + Slack
- P2: 邮件 + Slack
- P3: Slack

## 6. 仪表盘视图设计

### 6.1 视图分类
1. **概览视图 (Overview)**: 核心KPI卡片 + 实时趋势图
2. **性能视图 (Performance)**: 响应时间分布 + 成功率趋势
3. **成本视图 (Cost)**: Token使用 + 成本分析
4. **健康视图 (Health)**: 系统资源 + 服务状态
5. **告警视图 (Alerts)**: 活跃告警 + 告警历史

### 6.2 刷新频率
- 实时指标: 5秒
- 分钟级指标: 30秒
- 小时级指标: 5分钟
- 日级指标: 1小时
